## 핵심 구현 및 개발 포인트
- **Spring Security 권한 체계를 활용한 등급별 메뉴 제어**: 백엔드에 Spring Security를 적용하여 로그인한 사용자의 권한을 확인하도록 설정했습니다. 일반 사용자와 관리자의 접근 권한을 분리하고, 권한에 따라 화면에 보이는 메뉴가 다르게 나타나도록 제어하여 보안성을 높였습니다.
- **ERP 관리자 전용 통합 대시보드 구현**: 관리자가 시스템 현황을 한눈에 파악할 수 있도록 전용 대시보드 화면을 만들었습니다. 오늘 방문자 수, 총 회원 수, 실시간 접속자 및 트래픽 현황을 상단에 배치하고, 최근 가입한 회원 목록과 최근 생성된 공방의 승인 상태(승인/거절/대기)를 표 형태로 구성하여 관리 편의성을 높였습니다.
- **사용자 정보 관리를 위한 마이페이지 구현**: 각 계정이 본인의 세부 정보와 권한을 안전하게 확인하고 관리할 수 있도록 마이페이지를 구현했습니다. 로그인 세션을 안전하게 유지하는 설정을 바탕으로 사용자 정보를 화면단에 정확하게 전달하여, 개인 정보 확인 및 수정이 안정적으로 이루어지도록 처리했습니다.

## 트러블 슈팅
- **[현상]** 사용자가 로그인 후 오랜 시간 페이지를 방치하여 세션(인증)이 만료된 상태에서, 대시보드의 특정 예약 관리 기능을 클릭했을 때 화면이 멈추거나 백엔드 서버로부터 Internal Server Error (500) 또는 권한 에러가 발생하며 사용자 경험을 해치는 현상 발생.
- **[원인]** 세션이 만료되어 백엔드(Spring Security)가 접근 권한 에러를 반환했으나, 프론트엔드(React) 단에서 비동기 API 요청 시 서버의 에러 응답을 캐치하여 페이지를 이동시키거나 상태를 초기화하는 등의 예외 처리 로직이 누락되어 있었기 때문임.
- **[해결]** 모든 API 요청을 처리하는 코드를 하나로 묶어 '공통 에러 처리 로직'을 구현했습니다. 서버로부터 세션 만료나 권한 에러를 받는 즉시 화면단에서 이를 감지하게 만들었으며, 사용자에게 안내 창을 띄운 뒤 브라우저에 남아있던 이전 로그인 정보를 지우고 로그인 화면으로 자동 이동하도록 구현하여 사용자 이탈과 서버의 2차 예외 발생을 차단했습니다.
- **[배운점]** 전산 시스템은 정상적인 데이터 통신뿐만 아니라, 인증 만료와 같은 예외 상황이 발생했을 때 클라이언트 단에서 이를 명확하게 인지하고 사용자를 안전한 경로로 유도하는 예외 처리 설계가 사용자 경험과 시스템 신뢰도에 직결됨을 배웠습니다.
