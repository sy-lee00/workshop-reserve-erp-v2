>**통합 정보 관리 및 보안 권한 제어를 적용한 공방 운영 ERP 플랫폼 프로젝트**

## 개요
### **프로젝트 성격**: KH정보교육원 국비 팀 프로젝트(4인)  
### **주요 역할**: 전산 시스템 풀스택 개발 (React 기반 프론트엔드 아키텍처 구축 및 Spring Security 기반 권한/인증 로직 개발)
### **작업 기간**: 2025.11 ~ 2025.12(7주)

---

## 기술 스택

### **Backend & Database**
- **Framework**: Spring Boot 3.5.6, Spring Security, MyBatis
- **Language**: Java 21
- **DB/Tools**: MySQL, Gradle, GitHub, Lombok

---

### **Frontend**
- **Language**: HTML5, CSS3, JavaScript (ES6+)
- **Library/Environment**: React, Node.js, npm, Bootstrap 5

---

## 핵심 구현 및 개발 포인트
- **Spring Security 연동 등급별 권한 제어**: 백엔드의 Spring Security 권한(Role) 체계와 프론트엔드의 조건부 렌더링을 연동했습니다. 이를 통해 관리자 등급별(Super, CS, Payment)로 대시보드 내 접근 및 조작 권한을 동적으로 제어하고, 비인가 사용자의 접근을 차단하여 학사/행정 시스템과 유사한 수준의 다중 권한 보안 구조를 구현했습니다.
- **2중 데이터 검증을 통한 무결성 확보**: 회원가입 및 클래스 예약 신청 폼 작성 시, React를 활용한 프론트엔드 단의 실시간 정규표현식 검증과 백엔드 파라미터 검증을 동시 적용하여 유효하지 않은 데이터가 DB에 인입되는 것을 방지하고 데이터 무결성을 확보했습니다.
- **외부 오픈 API 연동 및 UX 커스텀**: Kakao Map API를 활용하여 공방 위치 데이터를 화면에 시각화하고, 비동기 통신을 통해 지도 내 커스텀 조작 이벤트를 제어하여 대시보드 내 타 기능과의 인터랙션 간섭을 최적화했습니다.

---

## 트러블 슈팅
- **[현상]** 사용자가 로그인 후 오랜 시간 페이지를 방치하여 세션(인증)이 만료된 상태에서, 대시보드의 특정 예약 관리 기능을 클릭했을 때 화면이 멈추거나 백엔드 서버로부터 Internal Server Error (500) 또는 권한 에러가 발생하며 사용자 경험을 해치는 현상 발생.
- **[원인]** 세션이 만료되어 백엔드(Spring Security)가 접근 권한 에러를 반환했으나, 프론트엔드(React) 단에서 비동기 API 요청 시 서버의 에러 응답을 캐치하여 페이지를 이동시키거나 상태를 초기화하는 등의 예외 처리 로직이 누락되어 있었기 때문임.
- **[해결]** 모든 API 요청을 처리하는 코드를 하나로 묶어 '공통 에러 처리 로직'을 구현했습니다. 서버로부터 세션 만료나 권한 에러를 받는 즉시 화면단에서 이를 감지하게 만들었으며, 사용자에게 안내 창을 띄운 뒤 브라우저에 남아있던 이전 로그인 정보를 지우고 로그인 화면으로 자동 이동하도록 구현하여 사용자 이탈과 서버의 2차 예외 발생을 차단했습니다.
- **[배운점]** 전산 시스템은 정상적인 데이터 통신뿐만 아니라, 인증 만료와 같은 예외 상황이 발생했을 때 클라이언트 단에서 이를 명확하게 인지하고 사용자를 안전한 경로로 유도하는 예외 처리 설계가 사용자 경험과 시스템 신뢰도에 직결됨을 배웠습니다.
