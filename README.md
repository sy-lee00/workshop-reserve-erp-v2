>**공방 예약의 편의성과 운영 효율성을 극대화한 통합 관리 플랫폼 프로젝트**

## 개요
### **프로젝트 성격**: KH정보교육원 국비 팀 프로젝트(4인)  
### **주요 역할**
#### **1. 데이터 기반의 이커머스 UX 최적화**
    - 마이페이지 및 개인화 영역: 위시리스트, 결제 내역 등 사용자 개인화 화면 담당
    - 인증 시스템: 회원가입 및 로그인 프로세스 UX 설계
#### **2. 효율적인 업무 환경을 위한 ERP 인터페이스 설계**
    - ERP 대시보드: 운영 관리자용 통계 시각화 및 권한 관리 시스템 구축 
### **작업 기간**: 2025.11 ~ 2025.12(7주)

## 핵심 구현 및 퍼블리싱 포인트
- **권한에 따른 유연한 관리자 대시보드 인터페이스**: 권한(Role)에 따라 UI 요소가 동적으로 제어되는 조건부 렌더링을 적용했습니다. 관리자 대시보드에서는 관리자 등급별(Super, CS, Payment)로 예약 승인/취소 등 복잡한 조작이 용이하도록 그리드 시스템을 최적화하고, 일반 사용자에게는 등급별(Workshop, User)로 예약 현황에 집중된 간결한 뷰를 제공하여 사용 주체별 최적화된 경험을 설계했습니다.
- **폼 유효성 검사 및 실시간 피드백**: 회원가입 및 클래스 예약 신청 시, 정규표현식과 React의 이벤트 핸들링을 결합하여 실시간 유효성 검사 기능을 구현했습니다. 사용자가 잘못된 값을 입력할 경우 즉각적인 시각적 피드백(경고 문구, 테두리 강조 등)을 제공하여 데이터의 무결성과 편의성을 동시에 확보했습니다.
- **카카오 맵 연동 및 지도 커스텀 디자인**: 사용자가 지도를 보며 공방 정보를 확인하고 즉시 예약할 수 있는 환경을 구축했습니다. 또한 'Ctrl + 스크롤' 조합으로만 지도를 확대/축소를 조작하게 설정하여, 일반적인 화면 탐색과 지도 조작이 겹치지 않도록 UX를 개선했습니다.

## 트러블 슈팅
- **[현상]** 프로젝트 규모가 커짐에 따라 서로 다른 컴포넌트 간의 CSS 클래스명 중복으로 인해 의도하지 않은 스타일 덮어쓰기 및 레이아웃 깨짐 현상 발생.
- **[원인]** 일반 CSS 사용 시 모든 스타일이 전역 범위로 적용되어, 컴포넌트 단위의 독립적인 스타일 유지가 어려움. 
- **[해결]** CSS Module을 도입하여, 컴포넌트별로 스타일 범위를 격리하여 충돌 문제를 근본적으로 해결했습니다.
- **[배운점]** 컴포넌트 기반 아키텍처에서는 스타일 또한 모듈화하여 관리하는 것이 유지보수성과 코드 안전성에 얼마나 중요한지 깨달았습니다.
